# 🚀 AI 代理网关核心模块开发提示词

## 🎯 角色设定
你现在是一位精通 Node.js 与高并发网络编程的资深后端架构师。我正在开发一个名为 `agent-hub-node` 的 AI 代理网站项目，现在需要你帮我设计并编写核心的 **HTTP 转发网关（Gateway）** 模块。

## 📋 核心需求

### 1. 极致转发效率（最高优先级）
- **纯原生实现**：必须使用 Node.js 原生的 `http` 模块进行请求和响应的双向转发，不依赖任何第三方 Web 框架。
- **强制使用字节流转发**：必须使用 Node.js 原生的 `stream`（流）机制。
- **严禁内存缓存**：绝对禁止将大模型的 SSE（Server-Sent Events）流式响应在内存中完整缓存后再发送。必须采用边接收边转发的字节流管道（Pipeline）模式。
- **零拷贝与低延迟**：采用非阻塞 I/O，确保在高并发下转发延迟极低，提升首字节响应速度（TTFB）。

### 2. 本地 SQLite 自动初始化与按分钟持久化统计
- **自动创建数据库**：程序启动时，需自动检测本地 SQLite 数据库文件是否存在。若不存在，则自动创建数据库文件及所需的统计表。
- **按分钟统计**：统计表需以“分钟”为维度进行汇总（包含 stat_time, prompt_tokens, completion_tokens, total_tokens, bytes_in, bytes_out 等字段）。每次写入时，若当前分钟的记录已存在，则进行累加更新（Upsert）；若不存在，则插入新记录。
- **数据拦截与解析**：在流式转发过程中，拦截并解析响应中的 `usage` 字段，同时统计入站和出站的字节数。
- **异步落盘**：将统计数据异步写入本地 SQLite 数据库，确保不影响主转发流的性能。

### 3. 定时清理与归档策略
- **自动清理**：为防止 SQLite 数据库无限膨胀，需实现一个定时任务（例如每小时执行一次）。
- **清理逻辑**：自动删除数据库中超过 30 天（时间跨度可配置）的分钟级统计记录，仅保留近期的历史数据。

### 4. WebSocket 实时监控推送
- **实时数据推送**：集成 WebSocket 功能，方便外部监控面板实时查看当前网关信息。
- **推送内容**：每当有新的请求完成并写入数据库后，通过 WebSocket 向所有连接的客户端广播最新的统计数据（如本次请求的 Token 消耗、字节数，以及当前分钟的累计统计）。
- **轻量级连接**：确保 WebSocket 连接不会阻塞主转发线程，保持网关的高性能。

### 5. 极简架构
- 整个网关只保留四个核心能力：**字节流转发**、**SQLite 按分钟数据落盘**、**定时自动清理**、**WebSocket 实时推送**。
- 不需要实现任何 RESTful 管理接口、限流控制或鉴权中间件。
- 保持代码极致精简，专注于核心功能。

## 🛠️ 技术与输出要求

- **技术栈**：Node.js 原生 `http` 模块 + `ws` 库 + 原生 JavaScript（不使用 TypeScript）+ SQLite（推荐使用 `better-sqlite3`）。
- **输出要求**：
  1. 给出基于原生 `http` 的核心转发逻辑代码实现（重点展示字节流转发和流式 Token 解析部分）。
  2. 给出 SQLite 数据库的自动初始化逻辑（建库、建表）与按分钟统计的异步写入（Upsert）逻辑。
  3. 给出定时清理过期数据的实现逻辑。
  4. 给出 WebSocket 服务的初始化及消息广播逻辑。
  5. 代码需包含清晰的中文注释，详细说明高并发与流式转发下的性能优化点。
